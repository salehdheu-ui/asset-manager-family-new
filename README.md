# صندوق العائلة — Family Fund OS

نظام متكامل لإدارة صندوق مالي عائلي: مساهمات شهرية، سلف بجدولة أقساط، مصروفات، توزيع رأس المال على طبقات محمية، حوكمة بوضع طوارئ، وسجل تدقيق دائم — بواجهة عربية مهيأة للجوال.

## البنية

| الطبقة | التقنية |
|---|---|
| الواجهة | React 19 + Vite + Tailwind + shadcn/ui (RTL) |
| الخادم | Express 5 + TypeScript |
| قاعدة البيانات | PostgreSQL + Drizzle ORM |
| الاختبارات | Vitest (حسابات مالية + قواعد أمان) |

## التشغيل محلياً

```bash
npm install
cp .env.example .env       # ثم عدّل القيم
npm run db:push            # إنشاء/تحديث الجداول (إضافات آمنة فقط)
npm run dev                # http://localhost:5000
```

## الأوامر

| الأمر | الوظيفة |
|---|---|
| `npm run dev` | تشغيل التطوير (خادم + واجهة معاً) |
| `npm run build` | بناء الإنتاج إلى `dist/` |
| `npm start` | تشغيل نسخة الإنتاج |
| `npm test` | تشغيل الاختبارات |
| `npm run check` | فحص أنواع TypeScript |
| `npm run db:push` | مزامنة مخطط قاعدة البيانات |

## المفاهيم الأساسية

- **الأدوار**: `admin` (الوصي — كل الصلاحيات) و`user` (عضو — يرى بياناته فقط ويقدم طلبات تنتظر الاعتماد).
- **طبقات رأس المال**: صافي الأصول يوزّع سنوياً على: محمي 45٪، طوارئ 15٪، مرن 20٪، نمو 20٪ (قابلة للتعديل من الإعدادات بشرط مجموع 100٪).
- **السلف**: طلب العضو يبدأ معلقاً؛ الاعتماد ينشئ جدول أقساط يمتص قسطه الأخير فرق التقريب. السلفة المسددة بالكامل تختفي من القائمة النشطة وتبقى في التقارير.
- **وضع الطوارئ**: من صفحة الحوكمة — يجمّد طلبات السلف والمساهمات للأعضاء مع بقاء صلاحيات الوصي.
- **سجل التدقيق**: كل عملية حساسة (اعتماد، حذف، تعديل، تغيير أمين، طوارئ) تُوثّق باسم منفذها ولا تُحذف.
- **النسخ الاحتياطي**: تلقائي يومي (عند تفعيله من الإعدادات) + يدوي، ويشمل كل الجداول بما فيها سجل التدقيق.

## ملاحظات نشر

- `SESSION_SECRET` إلزامي في الإنتاج.
- بعد كل تحديث يضيف أعمدة جديدة شغّل `npm run db:push` — الإضافات آمنة ولا تمس البيانات.
- CI (GitHub Actions) يفحص الأنواع والاختبارات والبناء وثغرات الاعتماديات على كل تحديث.
